# 单词记忆卡片应用实验报告

## 一、项目概述

### 1.1 项目背景

随着全球化的发展，英语学习变得越来越重要。传统的单词记忆方法效率低下，容易遗忘。间隔重复算法（Spaced Repetition System, SRS）是一种科学的记忆方法，通过根据记忆曲线调整复习时间，能够显著提高记忆效率。本项目基于Vue3开发了一个单词记忆卡片应用，采用简化的间隔重复算法，帮助用户高效记忆英语单词。

### 1.2 项目目标

1. 开发一个基于Vue3的单词记忆卡片应用
2. 实现简化的间隔重复算法，根据学习情况自动调整复习时间
3. 支持单词的添加、删除和管理功能
4. 提供可视化的学习进度跟踪
5. 实现流畅的卡片翻转动画
6. 适配不同屏幕尺寸，提供良好的移动端体验

### 1.3 项目范围

本项目为前端单页应用，使用本地存储持久化数据，无需后端支持。主要功能包括：
- 单词本管理（添加、删除、查看单词）
- 记忆测试（带有翻转动画的卡片）
- 学习进度跟踪
- 本地数据存储
- 简化的间隔重复算法

#### 3.1.2 数据结构设计

```javascript
const word = {
  id: Date.now(),           // 唯一标识
  word: 'example',          // 单词
  meaning: '例子',          // 释义
  nextReview: Date.now(),   // 下次复习时间
  interval: 1,              // 复习间隔（天）
  easiness: 2.5,            // 难度系数
  status: 'new'             // 状态：new, reviewed, failed
};
```

#### 3.1.3 核心算法实现

**简化的间隔重复算法**：

```javascript
export function updateWordStatus(id, remembered) {
  const words = getWords();
  const wordIndex = words.findIndex(word => word.id === id);
  if (wordIndex === -1) return words;

  const word = words[wordIndex];
  const now = new Date();
  
  if (remembered) {
    // 记住了，增加间隔
    word.interval = Math.round(word.interval * word.easiness);
    word.status = 'reviewed';
  } else {
    // 没记住，重置间隔
    word.interval = 1;
    word.status = 'failed';
  }

  // 设置下次复习时间
  const nextReview = new Date(now.getTime() + word.interval * 24 * 60 * 60 * 1000);
  word.nextReview = nextReview.toISOString();
  
  words[wordIndex] = word;
  saveWords(words);
  return words;
}
```

#### 3.1.4 实现难点与解决方案

**难点**：如何设计高效的本地存储方案，确保数据的一致性和可靠性

**解决方案**：
- 使用JSON格式存储数据，便于序列化和反序列化
- 每次更新数据时，重新保存整个单词列表
- 提供初始化数据，确保应用首次运行时有数据可用

#### 3.3.2 核心实现

**添加新单词**：

```vue
<template>
  <!-- 添加新单词表单 -->
  <div class="add-word-form">
    <input 
      v-model="newWord.word" 
      type="text" 
      placeholder="输入单词" 
      required
    >
    <input 
      v-model="newWord.meaning" 
      type="text" 
      placeholder="输入释义" 
      required
    >
    <button @click="addNewWord">添加单词</button>
  </div>
</template>

<script setup>
const addNewWord = () => {
  if (newWord.value.word.trim() && newWord.value.meaning.trim()) {
    addWord(newWord.value.word.trim(), newWord.value.meaning.trim());
    newWord.value = { word: '', meaning: '' };
    loadWords();
  }
};
</script>
```

#### 3.3.3 实现难点与解决方案

**难点**：如何实时更新单词列表

**解决方案**：
- 使用Vue的响应式系统
- 添加或删除单词后，重新加载单词列表
- 使用onMounted钩子在组件挂载时加载数据

## 五、项目总结与展望

### 5.1 项目成果

本项目成功开发了一个基于Vue3的单词记忆卡片应用，实现了所有预期功能：

1. **单词管理**：支持添加、删除和查看单词
2. **记忆测试**：带有流畅翻转动画的记忆卡片
3. **间隔重复算法**：根据学习情况自动调整复习时间
4. **进度跟踪**：可视化展示学习进度和统计数据
5. **响应式设计**：适配不同屏幕尺寸
6. **本地存储**：数据持久化，无需后端支持

### 5.2 技术亮点

1. **组件化开发**：将应用拆分为多个功能明确的组件，提高代码的可维护性和复用性
2. **流畅的卡片翻转动画**：使用CSS 3D变换实现流畅的卡片翻转动画
3. **简化的间隔重复算法**：根据学习情况智能调整复习时间
4. **响应式设计**：适配不同屏幕尺寸，提供良好的移动端体验
5. **本地数据存储**：使用LocalStorage实现数据持久化，无需后端支持

### 5.3 存在的问题与改进方向

1. **数据安全性**：LocalStorage存储在浏览器中，数据安全性较低
   - **改进方案**：添加数据加密功能，保护用户数据

2. **数据同步**：无法在多设备间同步数据
   - **改进方案**：添加云端同步功能，支持多设备使用

3. **单词分类**：不支持按类别管理单词
   - **改进方案**：添加单词分类功能，支持按类别管理和学习单词

4. **单词发音**：缺少单词发音功能
   - **改进方案**：集成语音合成API，添加单词发音功能

5. **学习统计**：统计数据不够丰富
   - **改进方案**：添加学习趋势图、学习时间统计等功能

### 5.4 未来展望

1. 添加更多交互功能，如单词拼写练习、例句展示等
2. 集成AI技术，提供个性化的学习建议
3. 支持导入/导出单词列表，方便用户管理数据
4. 添加社交功能，支持用户分享学习进度
5. 开发移动端APP，提供更好的移动端体验
